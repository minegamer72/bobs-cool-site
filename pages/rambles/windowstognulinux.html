Switching to GNU/Linux



Here are a few things that this page will assume:

  • You are a Microsoft Windows user.
  • You use very little to no free software or don't know.
  • You have little to no experience with a command line interface (like Windows Powershell or CMD)
  • You have no experience with GNU/Linux (or erroneously referred to as just Linux)

Before

I love checklists. So, I will provide one that goes over what I believe to be the most important parts before switching to GNU/Linux. We will discuss every one of these points in this section.

See if the software you currently use has a GNU/Linux version
(Optional but heavily recommended) Switch to libre alternatives of the software you use.
(Optional) Learn to use virtual machines
(Optional) Learn to dual boot
Find the right GNU/Linux distribution for you


Before switching to a GNU/Linux operating system, you should research the apps you use the most and see if they have a GNU/Linux version. Most of the time, there will be a GNU/Linux version. However, with software like those from the Adobe Creative Cloud, there will not be any. This leaves you with three options:
  1. Learn to use virtual machines
    2. (virtual machines aren't that hard!)
  2. Figure out how to get it to work under compatibility tools like WINE.
    3. (it is mostly just as easy as opening the app with the tool)
  3. Learn to use a libre alternative.
    4. (Note: Libre software is the same as free software. I will continue to use the term libre software.)

Before switching to GNU/Linux, you should learn about 1 and 3. The only caveat with virtual machines is that it requires extra computing power, however, the extra computing power required is usually not that much but more is always better.
Learning to use libre alternatives to software you already use may or may not be a challenge depending on the kind of person you are. This is why "Learn to use a libre alternative" is in Before
Here is a chart that contains libre alternatives to common proprietary software:

If you choose to switch software, you should take time to learn it to the same level as the proprietary one you were already using.
You also have the option of using virtual machines. I recommend that you use a libre virtual machine manager (I will use the term hypervisor) like QEMU and maybe not VirtualBox. QEMU uses the command line for its operation but you are able to get an app that uses a graphical window, although I recommend using the command line Powershell to get used to using the command line. I will demonstrate using QEMU and VirtualBox later.
