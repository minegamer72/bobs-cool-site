The Terminal



The terminal requires a long explanation, so this is going to be a long ramble. If you are looking for how to switch from Windows/MacOS to GNU/Linux, read this ramble.

The Terminal

This is one of the largest reasons that people don't switch to GNU/Linux is because they do not want to spend time or do not have the time to learn the terminal. Eventually, GNU/Linux distributions will get so streamlined that the terminal won't ever be needed. But for now, you should learn the basic terminal commands.
Open the terminal (you can search Terminal or Konsole for KDE) and type a few of these commands.

ls

This lists files in the current folder (or in this case, directory). You should see something similar to this:

[user@debian ~]$ ls
Desktop Downloads
Documents Videos
[user@debian ~]$

Congratulations! Instead of using the 'ls' command, you can open the file manager Dolphin or Thunar to view and manage files. You may notice the '~' in the prompt. That is called the "working directory" because it is the directory you are working in. '~' is an easy way to say "home directory" which is the equivalent to the windows user folder. We can get the path of the current working directory by typing the following command:

pwd

Note: pwd stands for "print working directory"

[user@debian ~]$ pwd
/home/user/
[user@debian ~]$

You may notice that file paths are completely different in GNU/Linux. Instead of having a drive letter followed by back slashes, you instead just have forward slashes just like website urls. As a result, it is a lot easier to type GNU/Linux directories than to type Windows directories.
C:\Users\user\ vs /home/user
If you have ever messed around with Windows, you have probably went to the C:\ directory. This is what is known as the "root directory" and in GNU/Linux it is known as '/'. We can go between directories by typing this command:

cd

Note: cd stands for "change directory"

[user@debian ~]$ cd /
[user@debian /]$

Then, if you type ls, you will see that the root directory is extremely different from Windows.
[user@debian ~]$ ls
bin dev home lib64 mnt proc run srv tmp var boot etc lib lost+found opt root sbin sys usr
[user@debian /]$

If you are following along, you can probably guess that typing "cd ~" will return you to the users home directory. You can also just type "cd" and it will default to the home directory.
If you want to go to the "Documents" directory, you can either type "cd /home/user/Documents" or if you are in the same directory as the Documents directory, you can just type "cd Documents/". Most terminals will have "tab completion" which will complete words in the given context.
[user@debian ~]$ cd Docu TAB
[user@debian ~]$ cd Documents/

This logic can also apply to commands. Anyway, now that you are in the Documents folder, lets learn how to make files.

touch

The touch command will create empty files with the given name. The name "touch" comes from the ability to literally touch a file, which updates the files modified and access timestamp.
[user@debian Documents]$ touch important document
[user@debian Documents]$ ls
document
important
[user@debian Documents]$

What happened here? Because the space is a seperator for arguments (basically telling a program what to do), it created two empty files "important" and "document". We can make touch treat this like one argument by surrounding it in quotation marks like this:
[user@debian Documents]$ touch "important document"
[user@debian Documents]$ ls
document
important
'important document'
[user@debian Documents]$

The quotation marks indicate a "string". This is just a programming term for a bunch of alphanumeric characters put between quotation marks. You probably don't want the "document" or the "important" files anymore, so we can remove them like this:

rm

rm stands for remove. It mainly uses the same kind of arguments like before, so we can either do:
[user@debian Documents]$ rm important
[user@debian Documents]$ rm document
[user@debian Documents]$ ls
[user@debian Documents]$
or
[user@debian Documents]$ rm important document
[user@debian Documents]$ ls
[user@debian Documents]$

If you are anything like me, you like to organize your Documents directory. We can do this by creating a new directory.

mkdir

This stands for make directory. The arguments are almost exactly like touch, meaning we can make the same mistake as before.
[user@debian Documents]$ mkdir important directory
[user@debian Documents]$ ls
directory
'important document'
important
[user@debian Documents]$

We can use the command "rmdir" in the same way to remove empty directories.
[user@debian Documents]$ rmdir important directory
[user@debian Documents]$ ls
'important document'
[user@debian Documents]$

If your directory has files in it, you will need to use the rm command with the -r option. The -r means "recursive" which means that it will go through every file and directory in that directory.
[user@debian Documents]$ rm -r important directory
[user@debian Documents]$ ls
'important document'
[user@debian Documents]$

We can then correct the mistake like before.
[user@debian Documents]$ mkdir "important directory"
[user@debian Documents]$ ls
'important directory'
'important document'
[user@debian Documents]$

Right now 'important document' is not in the 'important directory'. We can fix this using:

mv

We can move files using the mv command like this:
[user@debian Documents]$ mv important document important directory
mv: cannot stat 'important': No such file or directory
[user@debian Documents]$

What happened here? When using the mv command like in the example, it will first look at the arguments and treat every one of them like a file until it sees a directory. In this case, we never gave it a directory so it would error anyway. Not only that,
none of the supplied files exist, which is why we got the error we did. We can fix this by doing the same quotation marks method.
[user@debian Documents]$ mv "important document" "important directory"
[user@debian Documents]$ ls
'important directory'
[user@debian Documents]$

As a side note, it usually isn't a good idea to have spaces in your file names and directory names. You should also camelCase them. This way, it is easier to type in a terminal or reference in code. We can view the file like this:
[user@debian Documents]$ ls "important directory"
'important document'
[user@debian Documents]$

If you want to correct the file name, we can use the mv command to rename files and directories.
[user@debian Documents]$ mv "important directory" importantDirectory
[user@debian Documents]$ ls
importantDirectory
[user@debian Documents]$ mv importantDirectory/'important document' importantDocument
[user@debian Documents]$ ls
importantDirectory
importantDocument
[user@debian Documents]$

What happened here? The mv command renamed the file, but because we didn't tell it to move to the directory the file was in, it automatically moved to the current working directory. We can move it back into the directory the same way as before:
[user@debian Documents]$ mv importantDocument importantDirectory
[user@debian Documents]$ ls
importantDirectory
[user@debian Documents]$ ls importantDirectory
importantDocument
[user@debian Documents]$

For future reference, the correct command would have been: "mv importantDirectory/'important document' importantDirectory/importantDocument" We are going to cd into the importantDirectory and edit the importantDocument.
There are many text editors for GNU/Linux, but we're going to talk about the two big ones.
VIM NANO
  • Harder to learn
  • Modal text editor
  • Usually isn't included
  • Easier to learn
  • Non-standard controls
  • Usually included

I prefer the VIM editor because it allows me to get documents written faster. This page was written in VIM. You will probably need to install vim as it does not come with most GNU/Linux distributions. Let's learn package managment real fast:

apt

This is the package manager for debian. This also appears on Ubuntu based systems like Pop!OS. To install vim, and thus any package, you will need to use the argument "install" like this:
[user@debian Documents]$ apt install vim
...
E: Unable to acquire the dpkg frontend lock (/var/lib/dpkg/lock-frontend), are you root?
[user@debian Documents]$

What happened here? This is because we aren't "root". We can use the "su" command to switch to the root user. Think of this as the admin account for Windows.
[user@debian Documents]$ su root
Password: (input root password)
[root@debian Documents]#

We can now use the apt command.
[root@debian Documents]# apt install vim
Reading package lists... Done
Building dependency tree... Done
Reading state information... Done
...
After this operation, # MB of additional disk space will be used.
Do you want to continue? [Y/n] Y
...
[root@debian Documents]#

What if we don't want to switch users every time? We should now edit the /etc/sudoers file to include our user.
[root@debian Documents]# vim /etc/sudoers
## sudoers file.
##
## This file MUST be edited with the 'visudo' command as root.
## Failure to use 'visudo' may result in syntax or file permission errors
## that prevent sudo from running.
##
## See the sudoers man page for the details on how to write a sudoers file.
##

BEFORE YOU START TRYING TO TYPE!
VIM is a modal text editor, meaning we need to change modes. Right now, we are in "normal" mode. This is the preferred mode for moving about the editor. We use the keys h (left) j (down) k (up) l (right) for cursor movement. If this seems odd to you, the idea is that your fingers remain on the home row of the keyboard. "H is at the left, J swoops down, K points up, L is at the right". I could make up a better phrase, but since this works for me this is all I could bother making. To get into insert/edit mode, we need to either use i or a. A is for after, which starts inserting after the highlighted character. I is the same thing but it starts before. Think of it like this:
i|@|a
For organizational purposes, we need to go to the section where it says "User privilege specification". We can search and jump to this string using /:
##
## User privilege specification
##
root ALL=(ALL:ALL) ALL

/User privilege specification

As you can see, root has all sudo privileges. We can copy this line and paste it on the next line using "yank and paste". Press ENTER to move the cursor to the search selection. Move the cursor over the line that starts with "root". It doesn't matter where the cursor specifically is just yet. Press yy on your keyboard while in normal mode. This will copy it to the clipboard. While still in normal mode, move the cursor to the next blank line and press p. You should now have another "root" privelege.
Move the cursor over the r in root and press x until the word root is gone. Press i and type your username. It should now look something like this:
##
## User privilege specification
##
root ALL=(ALL:ALL) ALL
user ALL=(ALL:ALL) ALL

Press ESC to exit insert mode and type :wq to write/save and quit the file.
We can now exit the root user by typing "exit".
[root@debian Documents]# exit
[user@debian Documents]$

Now, we can run the command "sudo" to run a command once as the root user. You will use your root pasword.
[user@debian Documents]$ sudo whoami
[sudo] password for user:
root
[user@debian Documents]$
